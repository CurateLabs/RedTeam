# Operating Landscape Map

Multi-domain map of the operating environment. RedTeam's plain-language generalization of the Handbook v10 operating-environment framework (formerly taught as PMESII-PT in military contexts).

## Domains

Map factors and interactions across:

| Domain | What to scan |
|--------|----------------|
| **Power & governance** | Who decides, legitimizes, and enforces |
| **Security & risk** | Threats, safety, stability, coercion |
| **Economy & resources** | Money, supply, labor, scarcity |
| **Society & community** | Norms, identity, cohesion, grievance |
| **Information & narrative** | Media, rumor, data, persuasion |
| **Infrastructure & capabilities** | Systems, logistics, technology, capacity |
| **Physical environment** | Geography, climate, built environment |
| **Time & tempo** | Cycles, deadlines, history, momentum |

## When to use

- Complex situations with many interacting systems
- Geopolitical, market, or organizational political analysis
- Framing Critical Variables before an assumptions check

## Method

1. List key factors in each domain (not every domain needs equal depth).
2. Trace cross-domain links (e.g. information → society → power).
3. Avoid checkbox filling — look for feedback loops and second-order effects.
4. Hand off to `/redteam assumptions` or `/redteam futures` for stress-testing.

## Related commands

- `/redteam assumptions` — Critical Variables often map to these domains
- `/redteam frame` — reframing when the wrong domain dominates the narrative
- `/redteam futures` — scenario axes often drawn from domain uncertainties
